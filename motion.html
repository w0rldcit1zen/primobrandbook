07 · MOTION — PRIMO-MOTION

The system, in time.

primo-motion is Primo's Remotion pipeline for data videos and animated infographics. It reads dist/tokens.ts — the same tokens as everything else — so a colour or type change propagates to video with zero hand-copying. Motion itself is tokenized: durations come from the Fibonacci sequence in milliseconds, so pacing is as traceable as spacing.

MOTION TOKENS — THE RULES OF TIME
TokenValueUse
duration.xs130 msStagger step between siblings (bars, list items)
duration.sm210 msSecondary element entrances; accent-phrase delay
duration.md340 msSingle element build (a bar, a card)
duration.lg550 msHeadline reveal; scene element hero entrance
duration.xl890 msCount-ups; full-scene transitions
hold.min1300 msMinimum hold after any build before exit — legibility floor
easingcubic-bezier(0.22, 1, 0.36, 1)Decelerating ("ease-out-quint") for entrances; linear only for count-ups
Add to tokens.json (proposed canon). Motion values are not yet in the kit — add the block below, re-run build.py, and Remotion inherits them through tokens.ts: 
"motion": {
  "duration": { "xs": 130, "sm": 210, "md": 340, "lg": 550, "xl": 890 },
  "holdMin": 1300,
  "easing": "cubic-bezier(0.22, 1, 0.36, 1)",
  "why": "Fibonacci milliseconds — pacing as traceable as spacing. Hold floor protects legibility (WCAG 2.2.2 pause/stop thinking applies to autoplaying loops)."
}

Motion do-nots

  • Do not animate more than one counting number per scene.
  • Do not exit any text before its 1300ms hold.
  • Do not ease-in entrances — elements decelerate into place.
  • Do not flash: no element blinks more than three times per second (WCAG 2.3.1).
  • Do not autoplay with sound; captions on every voiced video.
  • Do not hand-copy hex into compositions — import from tokens.ts.
